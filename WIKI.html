Project Overview

This website is a public resource directory for Alameda County Probation and community partners. It helps users browse, search, filter, and match with appropriate community resources.

The site is mostly static, but it uses Vercel serverless API routes to read and update resources.json in GitHub. This keeps the public site simple while still allowing authorized staff to maintain the resource list.

Documentation cleanup: this wiki uses placeholders only. It does not include personal GitHub usernames, emails, repository URLs, passwords, tokens, or Vercel account details.

How it works

Layer Files Purpose
Frontend index.html, styles.css, translations.js Displays the public site, layouts, filters, Smart Match, and admin interface.
Data resources.json Stores all resource records used by the directory.
API api/login.js, api/resources.js, api/updateResources.js Checks admin password, reads GitHub JSON, and writes updates back to GitHub.
Deployment Vercel Hosts the static site and runs API routes.

Public Directory

Users can search and filter resources. Resource cards include key service details such as category, ages, description, contact, address, tags, and action link.

Smart Match

The wizard asks guided questions and displays matching resources based on the user's needs and selected factors.

Responsive Layouts

The site supports cards, list, accordion, detail/split, and table-style layouts. Mobile CSS prevents alternate layouts from creating page-wide horizontal scroll.

Accessibility

The stylesheet includes focus outlines, skip link support, high contrast improvements, reduced motion behavior, and an accessibility toolbar.

Admin Editing

Authorized users can add, edit, import, export, and save resources. Updates are sent to GitHub through backend API routes so credentials stay out of the browser.

Hidden Admin Access

The public Admin button is hidden to keep the interface clean. Admin access is still available through intentional triggers.

Method Trigger Best For
Desktop keyboard Ctrl + Shift + A Staff using desktop or laptop computers
URL trigger https://your-domain.com/?admin=1 Bookmarked admin access
Mobile / touch Tap the lower-right corner five times within three seconds Phones and tablets
Important: these triggers only open the login modal. The admin password is still checked by the backend using ADMIN_PASSWORD.

Vercel Environment Variables

Set these in Vercel Project Settings. Do not put real values inside source files or documentation.

Variable Required Placeholder Purpose
ADMIN_PASSWORD Yes <admin_password> Admin login password
GITHUB_TOKEN Yes <github_personal_access_token> Token used by backend API to read/write resources
GITHUB_OWNER Yes <github_owner_or_org> Owner or organization of the repository
GITHUB_REPO Yes <repository_name> Repository name
GITHUB_BRANCH Recommended main Branch containing the data file
GITHUB_JSON_PATH Recommended resources.json Path to the resource data file

Setup Workflow

1

Upload to GitHub

Place all project files in the repository, including api/, assets/, index.html, styles.css, and resources.json.

2

Create GitHub Token

Create a token limited to the repository with contents read/write access. Do not save the token in the source code.

3

Deploy to Vercel

Import the GitHub repository into Vercel. Add the environment variables listed above, then redeploy.

4

Test Public Site

Confirm search, filters, layouts, Smart Match, and mobile display work correctly.

5

Test Admin Save

Open the hidden admin panel, make a small edit, save it, and verify resources.json updated in GitHub.

Resource JSON Schema

Each resource should use consistent fields so search, filters, layouts, and Smart Match behave correctly.

{
  "name": "Example Resource Name",
  "category": "Education",
  "description": "Short public-facing description.",

  "contact": "Phone or email",
  "address": "Street address or service area",

Troubleshooting

Admin password not configured

Confirm ADMIN_PASSWORD exists in Vercel and redeploy.

Resources do not load

Check GITHUB_OWNER, GITHUB_REPO, GITHUB_BRANCH, GITHUB_JSON_PATH, and GITHUB_TOKEN.

Save fails

Confirm the token has contents read/write permission and has not expired.

Backup and Recovery

Because the data file is stored in GitHub, previous versions can be recovered from file history.

  1. Open the repository on GitHub.
  2. Open resources.json.
  3. View file history.
  4. Restore a previous version if needed.
  5. Refresh or redeploy the site.

Security Checklist

  • No GitHub token in source code.
  • No admin password in source code.
  • No personal GitHub username in documentation.
  • No personal email in documentation.
  • GitHub token is limited to the correct repository.
  • Vercel environment variables are configured before deployment.
  • Admin password is shared only with authorized staff.
  • Token is rotated if exposed or when staff access changes.
Never commit secrets: all credentials belong in Vercel Environment Variables, not in GitHub files.
